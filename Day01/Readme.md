# Introduction to Red Hat Enterprise Linux

## Why Should You Learn about Linux?

Linux is a critical technology for IT professionals to understand.

Linux is widely used worldwide. Internet users interact with Linux applications and web server systems daily by browsing the World Wide Web and using e-commerce platforms.

Linux is used far beyond the internet:
- Manages point-of-sale systems
- Powers stock markets globally
- Runs smart TVs and in-flight entertainment systems
- Operates most of the world’s top 500 supercomputers

Linux also powers:
- Cloud computing platforms
- Container-based microservices
- Software-defined storage
- Big data solutions

In modern data centers, Linux and Microsoft Windows are dominant operating systems. Linux adoption continues to grow across enterprise, cloud, and devices.

### Reasons to Learn Linux
- Interoperate with Linux systems from Windows environments
- Host applications and runtime environments
- Work with cloud computing platforms
- Support IoT and mobile applications
- Boost career opportunities (high demand for Linux skills)

- Modify and experiment with it
- Share improvements freely

This leads to rapid innovation and collaboration.

### Powerful Command-Line Interface (CLI)
- Full system control via CLI
- Enables automation, scripting, and remote administration
- Built-in design philosophy from the start

### Modular Design
- Components can be easily replaced or upgraded
- Suitable for both minimal systems and full workstations

## What Is Open Source Software?

Open source software allows users to:
- View source code
- Modify it
- Share and distribute it freely

### Key Concepts
- Source code = human-readable instructions
- Can be compiled or interpreted
- Governed by licenses

### Open Source vs Proprietary Software
| Feature | Open Source | Proprietary |
|--------|-------------|------------|
| Code Access | Available | Restricted |
| Modification | Allowed | Limited/Not allowed |
| Distribution | Free to share | Controlled |

### Benefits of Open Source
- **Control**: Modify and improve
- **Learning**: Study real-world code
- **Security**: Inspect and fix issues
- **Stability**: Not dependent on single vendor

---

## Types of Open Source Licenses

### 1. Copyleft Licenses
- Require derivatives to remain open source
- Promote sharing and improvements

**Examples:**
- GPL (GNU General Public License)
- LGPL

### 2. Permissive Licenses
- Allow reuse in proprietary software
- Focus on flexibility

**Examples:**
- MIT License
- BSD License
- Apache License 2.0

## What Is a Linux Distribution?

A Linux distribution (distro) is:
> A complete operating system built from the Linux kernel plus tools, libraries, and applications.

### Key Points
- Kernel created by Linus Torvalds (1991)
- Combined with GNU tools and other software
- Provides installation and package management

### Features of Distributions
- Include kernel + user programs
- Range from minimal to full-featured systems
- Provide updates and maintenance

---

## The Linux Kernel

The kernel is the core of Linux:
- Manages hardware
- Handles memory and processes
- Loads drivers

### History
- Created by Linus Torvalds in 1991
- Licensed under GPL
- Supported by global contributors including Red Hat

### Importance
- Powers servers, desktops, cloud systems, and devices
- Forms the base of all Linux distributions

---

## The Purpose of a Kernel

### Boot Process
1. System firmware loads bootloader
2. Bootloader loads kernel
3. Kernel initializes hardware
4. Services start (network, graphics, etc.)
5. Login prompt appears

### Interfaces
- **GUI (Graphical User Interface)**  
  - Visual, user-friendly
- **CLI (Command Line Interface)**  
  - Text-based, powerful, scriptable

---

## Overview of Linux Distributions

A Linux distribution is a customized version of Linux built for specific purposes.

### Examples

| Distribution | Description |
|-------------|------------|
| **RHEL** | Enterprise-focused, stable, long-term support |
| **CentOS** | Development platform for RHEL |
| **Fedora** | 최신 features, rapid releases |
| **Debian** | Stable, community-driven |

### Special Purpose Distros
- Raspberry Pi OS → lightweight for embedded devices
- Red Hat UBI → minimal cloud container base image

### Key Idea
Switching distributions is like switching smartphones:
- Core concepts remain the same
- Interface and tools may differ
